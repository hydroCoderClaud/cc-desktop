钉钉桥接配置指南

将钉钉机器人接入 Hydro Desktop，实现手机钉钉与桌面 Agent 双向对话

钉钉 飞书 企业微信 微信
📋 前提条件
  • Hydro Desktop 已安装并正常运行
  • 拥有钉钉组织的管理员权限（用于创建机器人应用）
⚠️ 重要限制 一个机器人只能绑定一个桌面端。同一组织内，同一个机器人的 AppKey/AppSecret 只能在一台 Hydro Desktop 上配置使用。
1 创建钉钉机器人应用
  1. 登录 钉钉开放平台
  2. 进入「应用开发」→「企业内部开发」→「创建应用」
  3. 填写应用名称和描述
  4. 创建完成后，在「凭证与基础信息」中获取 AppKey 和 AppSecret
2 配置机器人能力与权限
  1. 进入「添加应用能力」→ 添加「机器人」
  2. 消息接收模式选择 Stream 模式
  3. 在「权限管理」中开通以下权限：
    权限用途
    上传媒体文件图片转发到钉钉
    机器人发消息回复消息与主动发送
    qyapi_get_department_list桌面端拉取组织部门
    qyapi_get_department_member桌面端拉取组织成员
  4. 点击「发布应用」
📌 主动发送需要额外权限 如果希望从桌面端直接给钉钉组织成员发消息，上面四个权限缺一不可。修改权限后需重新发布应用。
3 在 Hydro Desktop 中配置
  1. 打开 Hydro Desktop，点击左下角齿轮图标，选择「钉钉设置」
  2. 填写以下配置：
字段说明
启用钉钉桥接打开开关
AppKey (ClientID)填入第一步获取的 AppKey。钉钉应用的凭证，用于鉴权和建立 Stream 连接
AppSecret (ClientSecret)填入第一步获取的 AppSecret，配合 AppKey 获取 access_token
机器人 Code (robotCode)填入钉钉机器人的编码，在「应用能力 → 机器人配置页」获取。
注意：robotCode 和 AppKey 是不同的标识
默认工作目录Agent 执行命令的工作目录（可选，强烈建议设置）
历史会话数量选择历史会话时显示的最大数量（默认 5，范围 1-20，在「高级设置」中配置）
  1. 点击「保存」，再点击「连接」
  2. 确认状态变为 已连接
4 验证配置
  1. 在钉钉中找到你的机器人，发送一条文本消息
  2. 观察 Hydro Desktop 是否出现钉钉会话并收到消息
  3. Agent 回复后，确认钉钉端能收到回复
建议第一次测试时在桌面端观察 Agent 回复过程，确认链路通畅后再让其他人使用。
⌨️ 命令参考

在钉钉中发送以 / 开头的消息，会被识别为命令：

命令说明
/help显示所有可用命令
/status查看当前会话状态与历史会话列表
/sessions列出当前聊天下的活跃会话
/new [目录]新建会话，可选指定工作目录
/resume [编号]恢复历史会话
/rename <名称>重命名当前会话
/close [编号]关闭当前或指定会话
❓ 常见问题
连接失败确认 AppKey/AppSecret 正确、应用已发布、消息接收模式为 Stream 模式、网络能访问钉钉服务器。
消息无回复确认 Hydro Desktop 状态显示「已连接」、API 配置正常。
桌面端主动发送列表为空确认 robotCode 已填写、通讯录权限已开通且应用已重新发布。
图片无法转发确认已开通「上传媒体文件」权限、图片大小不超过 20MB。
