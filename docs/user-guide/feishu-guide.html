飞书桥接配置指南

将飞书机器人接入 Hydro Desktop，实现手机飞书与桌面 Agent 双向对话，支持交互卡片命令

钉钉 飞书 企业微信 微信
📋 前提条件
  • Hydro Desktop 已安装并正常运行
  • 拥有飞书组织的管理员权限（用于创建企业自建应用）
⚠️ 重要限制 一个飞书应用只能绑定一个桌面端。多台桌面端同时连接同一个应用会导致消息丢失或重复回复。
1 创建飞书企业自建应用
  1. 登录 飞书开放平台
  2. 进入「开发者后台」→「创建企业自建应用」
  3. 填写应用名称和描述
  4. 创建完成后，在「凭证与基础信息」中获取 App ID 和 App Secret
2 配置机器人能力与权限
  1. 进入「添加应用能力」→ 添加「机器人」
  2. 在「权限管理」中开通以下权限：
    权限用途
    获取用户信息解析消息发送者名称
    获取通讯录信息桌面主动发送时选择联系人
    发送消息回复消息与主动推送
    上传图片转发图片到飞书
    读取图片资源接收飞书端发送的图片
  3. 发布应用并等待管理员审批通过
📌 连接方式说明 飞书桥接使用 WebSocket 长连接方式接收消息，无需配置 HTTP 请求 URL。
3 在 Hydro Desktop 中配置
  1. 打开 Hydro Desktop，点击左下角齿轮图标，选择「飞书设置」
  2. 填写以下配置：
字段说明
启用飞书桥接打开开关
App ID填入第一步获取的 App ID
App Secret填入第一步获取的 App Secret
默认工作目录Agent 执行命令的工作目录（可选，强烈建议设置）
历史会话数量选择历史会话时显示的最大数量（默认 5，范围 1-20）
  1. 点击「保存」，再点击「连接」
  2. 确认状态变为 已连接
4 验证配置
  1. 在飞书中找到你的机器人，发送一条文本消息
  2. 观察 Hydro Desktop 是否出现飞书会话并收到消息
  3. 发送 /help，确认能收到交互卡片命令菜单
建议第一次测试时在桌面端观察 Agent 回复过程，确认链路通畅后再让其他人使用。
⌨️ 命令参考

飞书使用交互卡片展示命令结果。发送以 / 开头的消息触发命令：

命令说明
/help显示可用命令（卡片形式）
/status查看系统状态
/sessions列出当前聊天下的活跃会话
/new [目录]新建会话
/resume [编号]恢复历史会话
/rename <名称>重命名当前会话
/close [编号]关闭当前或指定会话

卡片发送失败时会自动回退到纯文本菜单。

❓ 常见问题
连接失败确认 App ID/Secret 正确、应用已发布并通过审批、已添加机器人能力。
消息无回复确认 Hydro Desktop 状态显示「已连接」、API 配置正常。
图片无法转发确认已开通「上传图片」权限、图片大小不超过 20MB。
桌面端看不到可发送成员确认已开通「获取通讯录信息」权限、应用已发布。
