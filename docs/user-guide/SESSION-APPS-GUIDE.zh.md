# 会话应用使用指南

本文档面向普通使用者，说明什么是“会话应用”、适合拿它做什么，以及现在如何创建、修改和启动它。

## 什么是会话应用

会话应用可以理解成：

- 把一个已经打磨好的 Agent 工作流
- 沉淀成一个可以反复启动的专用会话入口

它不是复杂前端应用，也不是独立程序。

它更像一个“专用智能体会话模板”：

- 带自己的目标说明
- 带自己的系统提示词
- 带自己的启动消息
- 带自己的默认工作目录

当你再次启动它时，桌面会直接创建一个新的专用会话，而不用你每次从零重新描述。

## 适合哪些场景

会话应用适合这类需求：

- 经常重复做一类工作
- 这类工作有固定目标和规则
- 不需要复杂页面
- 主要通过对话推进

例如：

- 周报整理助手
- 会议纪要助手
- 需求拆解助手
- 文档审校助手
- 水文资料整理助手

## 和普通会话有什么区别

普通会话更适合：

- 临时问题
- 一次性任务
- 开放探索

会话应用更适合：

- 已经明确的一类固定工作
- 希望以后反复启动
- 希望每次都有统一开场和统一规则

简化理解就是：

- 普通会话：随手聊
- 会话应用：把某类常用工作沉淀下来反复用

## 现在可以怎么创建

当前有两种主要方式。

### 方式一：在聊天里创建

你可以：

- 点击聊天工具栏里的会话应用按钮
- 或输入 `/session-app`

然后桌面会在聊天里生成一张“会话应用配置卡片”。

你可以直接在卡片里修改：

- 应用名称
- 应用说明
- 启动消息模板
- 系统提示词
- 默认工作目录

确认时有两种结果：

- `仅保存到会话应用`
- `立即创建应用会话`

这适合你在一次真实对话里，边讨论边把一个能力提炼出来。

### 方式二：在能力管理里创建

入口：

- `能力管理 -> 会话应用`

这里适合做集中管理，可以：

- 新建应用
- 编辑应用
- 复制应用
- 删除应用
- 查看关联会话
- 直接启动新会话

## 这些字段分别是什么意思

### 应用名称

这就是应用在列表里的名字。

### 应用说明

帮助你自己或其他人理解这个应用是干什么的。

### 启动消息模板

每次启动这个应用时，桌面会自动把它作为第一条消息发给 Agent。

适合写：

- 每次启动都要做的开场任务
- 标准输入说明
- 固定的执行请求

### 系统提示词

这是隐藏在上下文里的规则，不会直接显示成聊天消息。

适合写：

- 目标限定
- 规则边界
- 执行偏好
- 应优先调用哪些 skill、CLI、MCP

### 默认工作目录

这是这个应用的基础工作目录。

需要注意两点：

1. 应用里保存的是基础目录。
2. 每次真正启动新会话时，桌面会在这个基础目录下再创建一个随机子目录。

所以不会所有运行实例都挤在同一个目录里。

## 默认工作目录现在怎么处理

如果你新建应用时没有手动指定目录，系统会自动给一个默认值。

默认规则是：

- 使用全局 Agent 输出目录
- 再拼上 `sessionapp`

例如：

- Windows：`C:/Users/你的用户名/cc-desktop-agent-output/sessionapp`
- macOS：`/Users/你的用户名/cc-desktop-agent-output/sessionapp`

真正启动新会话时，会再生成一层子目录，例如：

- `.../sessionapp/conv-a1b2c3d4`

## 启动后会自动带上什么

启动会话应用后，新会话会自动带上：

- 当前应用的系统提示词
- 当前应用的默认工作目录
- 当前应用的启动消息模板

所以从体感上看，它会比普通新会话更“有预设”。

## 修改应用后会发生什么

当前版本里，一个应用只有一份当前定义。

这意味着：

- 你修改后，新启动的应用会话会用最新定义
- 旧的应用会话如果之后继续使用，也会跟随当前定义

当前不会为历史会话单独锁一份旧快照。

这样做的好处是简单直接，代价是旧会话继续使用时也会受到应用修改影响。

## 删除应用后会发生什么

删除应用时：

- 应用定义会被删除
- 历史会话不会被删除

如果这个应用已经关联了一些历史会话，删除弹窗会提醒你慎重操作。

你可以把它理解成：

- 删除的是“应用入口”
- 不是“已经产生的聊天记录”

## 左侧会话列表怎么看

如果某个会话来自会话应用：

- 左侧会话列表会带会话应用图标
- 鼠标悬停可看到对应应用信息
- 点击应用图标可直接打开该应用的管理详情

这样你可以一边看历史会话，一边快速定位它对应的应用定义。

## 适合先怎么用

如果你第一次尝试，建议这样开始：

1. 先在普通 Agent 会话里把一类工作聊顺。
2. 感觉这个流程以后还会反复用，就点会话应用按钮或输入 `/session-app`。
3. 把名称、说明、启动消息和规则补完整。
4. 先保存，再多启动几次验证体感。
5. 之后逐步把它变成你自己的常用专用会话入口。

## 当前最重要的理解

会话应用不是要替代所有聊天。

它的价值是：

- 把常用工作流沉淀下来
- 让桌面里的 Agent 从“每次重新讲需求”
- 变成“直接启动一个已经定义好的专用会话”

如果你只是临时问个问题，用普通会话更合适。

如果你已经形成了一类稳定工作方式，会话应用就很合适。
