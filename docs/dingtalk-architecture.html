💡 关键设计点

  • 零公网依赖：Stream 模式由本地主动建立 WebSocket 连接到钉钉云，不需要公网 IP、域名、内网穿透
  • 复用现有架构：核心对话逻辑完全复用 AgentSessionManager + Claude SDK，DingTalk Bridge 只是新的 "入口"
  • 新增模块约 200-300 行：DingTalk Bridge + 会话映射 + 消息适配，符合项目小模块原则
  • 配置项：在 Hydro Desktop 设置中添加钉钉 AppKey / AppSecret / 开关，用户可随时启停
  • 前提条件：需要在钉钉开发者后台创建企业内部应用（免费），获取 AppKey + AppSecret
  • 局限：Hydro Desktop 必须保持运行；流式输出无法实时推送（钉钉不支持），只能攒完后一次发送
